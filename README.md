## 1. יצירת Google Sheet

1. היכנסו ל-[Google Sheets](https://sheets.google.com) עם **חשבון Google ארגוני** (מומלץ, לא חשבון פרטי בסביבת ייצור).
2. לחצו על **"+ גיליון אלקטרוני ריק"**.
3. שנו את שם הגיליון (לשונית למטה) ל-`תגובות`.
4. העתיקו את **מזהה הגיליון** מה-URL:

   ```
   https://docs.google.com/spreadsheets/d/  <<<SHEET_ID>>>  /edit
   ```

   המזהה הוא המחרוזת הארוכה בין `/d/` ל-`/edit`.

---

## 2. פתיחת Apps Script והדבקת הקוד

1. בתוך הגיליון לחצו: **תוספות → Apps Script** (Extensions → Apps Script).
2. מחקו את כל הקוד שנמצא בחלון `Code.gs`.
3. פתחו את הקובץ `apps-script/Code.gs` מהפרויקט הזה.
4. **הדביקו את כל התוכן** לתוך חלון Apps Script.
5. מצאו את השורה:

   ```javascript
   var SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID_HERE';
   ```

6. החליפו `YOUR_SPREADSHEET_ID_HERE` במזהה הגיליון שהעתקתם בשלב 1.
7. לחצו **שמור** (Ctrl+S / Cmd+S).

---

## 3. פריסת ה-Web App

1. בחלון Apps Script לחצו: **פרוס → פריסה חדשה** (Deploy → New deployment).
2. לחצו על סמל ההגדרות (גלגל שיניים) ובחרו **Web App**.
3. הגדרות:
   - **תיאור:** סקר אוטומציה
   - **בצע בשם (Execute as):** Me (אני)
   - **מי יכול לגשת (Who has access):**
     - לסביבה ארגונית פנימית: `Anyone within [your organization]`
     - לגישה ציבורית: `Anyone`
4. לחצו **Deploy**.
5. אשרו הרשאות גישה אם מתבקשים.
6. **העתיקו את ה-URL** שמופיע בחלון "Web App URL" — נראה כך:

   ```
   https://script.google.com/macros/s/AKfycby.../exec
   ```

> **חשוב:** בכל פעם שמשנים את הקוד ב-Apps Script יש ליצור **פריסה חדשה** (New deployment) ולא לעדכן קיימת, כדי שהשינויים ייכנסו לתוקף.

## 4. הגדרת כתובת ה-Script בפרויקט

1. פתחו את הקובץ `script.js` בתיקיית הפרויקט.
2. מצאו את השורה הראשונה:

   ```javascript
   const GOOGLE_SCRIPT_URL = 'YOUR_GOOGLE_SCRIPT_URL_HERE';
   ```

3. החליפו `YOUR_GOOGLE_SCRIPT_URL_HERE` ב-URL שהעתקתם בשלב 3.
4. שמרו את הקובץ.

## 5. פרסום ב-GitHub Pages

### שלב א' – יצירת Repository

1. היכנסו ל-[GitHub](https://github.com) וצרו חשבון אם עדיין אין לכם.
2. לחצו **New repository**.
3. שם: `automation-survey` (או כל שם שתרצו).
4. בחרו **Public** (חובה לשימוש חינמי ב-GitHub Pages).
5. לחצו **Create repository**.

### שלב ב' – העלאת הקבצים

**אפשרות A – ממשק ה-Web:**
1. בתוך ה-repository לחצו **Add file → Upload files**.
2. גררו את הקבצים: `index.html`, `styles.css`, `script.js`.
3. לחצו **Commit changes**.

**אפשרות B – Git (אם מותקן):**
```bash
git init
git add index.html styles.css script.js
git commit -m "Initial survey"
git branch -M main
git remote add origin https://github.com/<your-username>/automation-survey.git
git push -u origin main
```

### שלב ג' – הפעלת GitHub Pages

1. ב-repository לחצו **Settings**.
2. בתפריט הצד לחצו **Pages**.
3. תחת **Source** בחרו: `Deploy from a branch`.
4. Branch: **main**, תיקייה: **/ (root)**.
5. לחצו **Save**.
6. המתינו 1-2 דקות. הכתובת תופיע:

   ```
   https://<your-username>.github.io/automation-survey/
   ```

## 6. בדיקת הטופס

1. פתחו את כתובת ה-GitHub Pages בדפדפן.
2. מלאו את הטופס עם נתוני בדיקה.
3. לחצו "שליחת הסקר".
4. בדקו שמופיעה הודעת הצלחה.
5. פתחו את Google Sheet ובדקו שנוספה שורה חדשה עם כל הנתונים.

**בעיות נפוצות:**

| בעיה | פתרון |
|------|--------|
| לא מגיעים נתונים לגיליון | ודאו שה-URL ב-`script.js` תואם בדיוק ל-Web App URL |
| שגיאת הרשאות | פרסו מחדש את ה-Web App ואשרו הרשאות |
| הגיליון ריק (אין כותרות) | הכותרות נוצרות אוטומטית עם הגשה ראשונה |
| שגיאה "SPREADSHEET_ID not found" | ודאו שהמזהה ב-`Code.gs` נכון ושיש לכם גישה לגיליון |

## 7. ייצוא נתונים ועדיפות תהליכים

### ייצוא לאקסל

1. ב-Google Sheets לחצו: **קובץ → הורד → Microsoft Excel (.xlsx)**.

### סינון ועדיפות לפי דירוגים

הגיליון כולל עמודות דירוג (1-5) לכל תהליך:

- **דירוג ידניות** – כמה התהליך ידני
- **דירוג זמן** – כמה הוא גוזל זמן
- **דירוג טעויות** – שכיחות שגיאות
- **דירוג דחיפות** – כמה דחוף לשפר
- **דירוג מושפעים** – כמה אנשים מושפעים

**טיפ לעדיפות:** הוסיפו עמודת חישוב בגיליון:
```
=AVERAGE(AF2:AJ2)
```
זה ייתן ציון ממוצע לכל תהליך לצורך מיון.

לחצו **נתונים → מיון** לפי עמודת הציון הממוצע (מהגבוה לנמוך) כדי לזהות את התהליכים הדחופים ביותר.

## 8. הערות אבטחה

> **קראו לפני הפרסום בסביבת ייצור.**

- **אל תאספו מידע רגיש ביותר** (סיסמאות, מספרי זהות, מידע רפואי) בטופס זה ללא אישור מחלקת אבטחת מידע / IT.
- **אל תעלו קבצים חסויים** לשום שירות שאינו מאושר ארגונית.
- **הנתונים נשמרים ב-Google Sheets** ולא ב-GitHub. וודאו שהגיליון מוגבל לאנשים המורשים בלבד (שיתוף → מי יש לו גישה).
- **ה-Web App URL הוא ציבורי** — כל אחד שיודע אותו יכול לשלוח נתונים. לא מומלץ לסמוך על ה-SHARED_TOKEN כאמצעי אבטחה מלא.
- **לסביבת ייצור בחברה גדולה:** קבלו אישור IT/אבטחת מידע לפני הפצה. שקלו שימוש ב-"Anyone within [your organization]" ב-Web App ובגיליון.
- **השתמשו בחשבון Google ארגוני** ולא בחשבון גוגל פרטי לניהול הגיליון בסביבת ייצור.

## 9. מבנה הפרויקט

```
automation-survey/
├── index.html          # טופס הסקר המלא (9 חלקים)
├── styles.css          # עיצוב RTL עברי
├── script.js           # לוגיקה, וידוא, שליחה ל-Google Script
├── apps-script/
│   └── Code.gs         # Google Apps Script – שמירה ב-Google Sheets
└── README.md           # הוראות הגדרה ופריסה
```

### קבצים שצריך להעלות ל-GitHub Pages:
- `index.html`
- `styles.css`
- `script.js`

**לא צריך להעלות** את תיקיית `apps-script/` — הקוד שם רץ ב-Google Apps Script, לא על השרת.
