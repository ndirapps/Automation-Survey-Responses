## 1. יצירת Google Sheet

1. היכנסו ל-[Google Sheets](https://sheets.google.com) עם **חשבון Google ארגוני** (מומלץ, לא חשבון פרטי בסביבת ייצור).
2. לחצו על **"+ גיליון אלקטרוני ריק"**.
3. שנו את שם הגיליון (לשונית למטה) ל-`תשובות הסקר` (גם אם תדלגו על השלב הזה, ה-Script ייצור אוטומטית לשונית בשם הזה בהגשה הראשונה).
4. העתיקו את **מזהה הגיליון** מה-URL:

   ```
   https://docs.google.com/spreadsheets/d/  <<<SHEET_ID>>>  /edit
   ```

   המזהה הוא המחרוזת הארוכה בין `/d/` ל-`/edit`.

## 2. פתיחת Apps Script והדבקת הקוד

1. בתוך הגיליון לחצו: **תוספות → Apps Script** (Extensions → Apps Script).
2. מחקו את כל הקוד שנמצא בחלון `Code.gs`.
3. פתחו את הקובץ `apps-script/Code.gs` מהפרויקט הזה.
4. **הדביקו את כל התוכן** לתוך חלון Apps Script.
5. מצאו את השורה:

   ```javascript
   var SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID_HERE';
   ```

6. החליפו `YOUR_SPREADSHEET_ID_HERE` במזהה הגיליון שהעתקתם בשלב 1.
7. לחצו **שמור** (Ctrl+S / Cmd+S).

---

## 3. פריסת ה-Web App

1. בחלון Apps Script לחצו: **פרוס → פריסה חדשה** (Deploy → New deployment).
2. לחצו על סמל ההגדרות (גלגל שיניים) ובחרו **Web App**.
3. הגדרות:
   - **תיאור:** סקר אוטומציה
   - **בצע בשם (Execute as):** Me (אני)
   - **מי יכול לגשת (Who has access):**
     - לסביבה ארגונית פנימית: `Anyone within [your organization]`
     - לגישה ציבורית: `Anyone`
4. לחצו **Deploy**.
5. אשרו הרשאות גישה אם מתבקשים.
6. **העתיקו את ה-URL** שמופיע בחלון "Web App URL" — נראה כך:

   ```
   https://script.google.com/macros/s/AKfycby.../exec
   ```

> **חשוב:** בכל פעם שמשנים את הקוד ב-Apps Script יש ליצור **פריסה חדשה** (New deployment) ולא לעדכן קיימת, כדי שהשינויים ייכנסו לתוקף.

## 4. הגדרת כתובת ה-Script בפרויקט

1. פתחו את הקובץ `script.js` בתיקיית הפרויקט.
2. מצאו את השורה הראשונה:

   ```javascript
   const GOOGLE_SCRIPT_URL = 'YOUR_GOOGLE_SCRIPT_URL_HERE';
   ```

3. החליפו `YOUR_GOOGLE_SCRIPT_URL_HERE` ב-URL שהעתקתם בשלב 3.
4. שמרו את הקובץ.

## 5. פרסום ב-GitHub Pages

### שלב א' – יצירת Repository

1. היכנסו ל-[GitHub](https://github.com) וצרו חשבון אם עדיין אין לכם.
2. לחצו **New repository**.
3. שם: `automation-survey` (או כל שם שתרצו).
4. בחרו **Public** (חובה לשימוש חינמי ב-GitHub Pages).
5. לחצו **Create repository**.

### שלב ב' – העלאת הקבצים

**אפשרות A – ממשק ה-Web:**
1. בתוך ה-repository לחצו **Add file → Upload files**.
2. גררו את הקבצים: `index.html`, `styles.css`, `script.js`.
3. לחצו **Commit changes**.

**אפשרות B – Git (אם מותקן):**
```bash
git init
git add index.html styles.css script.js
git commit -m "Initial survey"
git branch -M main
git remote add origin https://github.com/<your-username>/automation-survey.git
git push -u origin main
```

### שלב ג' – הפעלת GitHub Pages

1. ב-repository לחצו **Settings**.
2. בתפריט הצד לחצו **Pages**.
3. תחת **Source** בחרו: `Deploy from a branch`.
4. Branch: **main**, תיקייה: **/ (root)**.
5. לחצו **Save**.
6. המתינו 1-2 דקות. הכתובת תופיע:

   ```
   https://<your-username>.github.io/automation-survey/
   ```

## 6. בדיקת הטופס

1. פתחו את כתובת ה-GitHub Pages בדפדפן.
2. מלאו את הטופס עם נתוני בדיקה.
3. לחצו "שלח סקר".
4. בדקו שמופיעה הודעת הצלחה.
5. פתחו את Google Sheet ובדקו שנוספה שורה חדשה עם כל הנתונים.

### בדיקת מספר תהליכים באותו סקר ("הוסף תהליך נוסף")

אותו עובד מאותה מחלקה יכול להגיש כמה תהליכים בהגשה אחת, בלי למלא מחדש את הפרטים הכלליים:

1. מלאו את הפרטים הכלליים (שלב 1) — לפחות "שם המחלקה".
2. עברו לשלבי התהליך (שלבים 2-9) ומלאו תהליך ראשון, כולל כל השדות החובה.
3. בשלב האחרון (סיכום) לחצו על "הוסף תהליך נוסף".
4. ודאו שמופיעה הודעת אישור קצרה, שדות התהליך מתאפסים, אך הפרטים הכלליים (מחלקה וכו') נשארים כפי שמולאו.
5. ודאו שמופיע תקציר "נוסף תהליך אחד לסקר" / "נוספו X תהליכים לסקר" עם שם התהליך שנוסף.
6. מלאו תהליך שני ולחצו "שלח סקר".
7. ודאו שמופיעה הודעת הצלחה עם מספר התהליכים שנשמרו (למשל "נשמרו 2 תהליכים").
8. פתחו את Google Sheet ("תשובות הסקר") וודאו שנוספו **שתי שורות נפרדות** — אחת לכל תהליך — כשבשתיהן אותם פרטי מחלקה/ממלא הסקר, ושעמודות "מספר תהליך בסקר" ו-"סה״כ תהליכים בסקר" מציגות 1/2 ו-2/2 בהתאמה.

### בדיקת מילוי סקר שני (אחרי "מלא סקר נוסף")

חשוב לבדוק גם את הזרימה המלאה של מילוי סקר נוסף באותו טעינת עמוד, בלי לרענן:

1. מלאו ושלחו סקר ראשון, עד הופעת מסך ההצלחה.
2. לחצו על הכפתור "מלא סקר נוסף".
3. ודאו שהטופס חוזר לשלב 1, **כל** השדות ריקים (כולל הפרטים הכלליים), שדות "אחר" מוסתרים, ושתקציר התהליכים התאפס.
4. מלאו את הטופס בשנית עם נתוני בדיקה אחרים.
5. הגיעו לשלב האחרון ולחצו "שלח סקר".
6. ודאו שכפתור השליחה מציג מצב טעינה רגעי (לא תקוע) ושמופיעה הודעת הצלחה.
7. פתחו את Google Sheet ("תשובות הסקר") וודאו שנוספה **שורה נוספת**, נפרדת מהשורות הקודמות, עם הנתונים של הסקר השני.

**בעיות נפוצות:**

| בעיה | פתרון |
|------|--------|
| לא מגיעים נתונים לגיליון | ודאו שה-URL ב-`script.js` תואם בדיוק ל-Web App URL |
| שגיאת הרשאות | פרסו מחדש את ה-Web App ואשרו הרשאות |
| הגיליון ריק (אין כותרות) | הכותרות נוצרות אוטומטית עם הגשה ראשונה |
| שגיאה "SPREADSHEET_ID not found" | ודאו שהמזהה ב-`Code.gs` נכון ושיש לכם גישה לגיליון |

## 7. ייצוא נתונים ועדיפות תהליכים

### ייצוא לאקסל

1. ב-Google Sheets לחצו: **קובץ → הורד → Microsoft Excel (.xlsx)**.

### סינון ועדיפות לפי דירוגים

הגיליון כולל עמודות דירוג (1-5) לכל תהליך (עמודות AS–AW):

- **דירוג - כמה התהליך ידני**
- **דירוג - כמה התהליך גוזל זמן**
- **דירוג - כמה שגיאות מתרחשות**
- **דירוג - דחיפות השיפור**
- **דירוג - כמות אנשים מושפעים**

**טיפ לעדיפות:** הוסיפו עמודת חישוב בגיליון:
```
=AVERAGE(AS2:AW2)
```
זה ייתן ציון ממוצע לכל תהליך לצורך מיון.

> אם תוסיפו או תסירו שאלות מהסקר בעתיד, ייתכן שמיקום העמודות ישתנה — בדקו מול שורת הכותרות בפועל בגיליון לפני שימוש בנוסחה.

לחצו **נתונים → מיון** לפי עמודת הציון הממוצע (מהגבוה לנמוך) כדי לזהות את התהליכים הדחופים ביותר.

## 8. דשבורד תיעדוף אוטומטי

ה-Script כולל דשבורד מובנה שמתעדכן אוטומטית ומסכם את התשובות לפי עדיפות לאוטומציה.

### מה נמצא בדשבורד

בכל פעם שהדשבורד מתעדכן, נוצרת (או מתעדכנת) לשונית בשם **"דשבורד תיעדוף"** עם:

1. **סיכום לפי מחלקה** – כמות תשובות, ממוצעי הדירוגים וציון תיעדוף כולל לכל מחלקה.
2. **סיכום לפי תהליך** – אותו פירוט, לכל שם תהליך.
3. **סיכום לפי מחלקה + תהליך** – אותו פירוט, לכל שילוב ייחודי של מחלקה ותהליך.
4. ארבעה תרשימים: מחלקות מובילות, תהליכים מובילים, שילובי מחלקה+תהליך מובילים, ופירוט דירוגים ממוצעים לפי מחלקה.

כל הטבלאות ממוינות מהציון הגבוה לנמוך, כך שהשורה הראשונה היא תמיד התהליך/המחלקה עם העדיפות הגבוהה ביותר לאוטומציה.

**ציון התיעדוף** מחושב כך (כל הממוצעים הם על סקאלה 1-5):

```
ציון = ממוצע ידניות * 0.25
     + ממוצע זמן    * 0.25
     + ממוצע שגיאות * 0.20
     + ממוצע דחיפות * 0.20
     + ממוצע השפעה על אנשים * 0.10
```

### עדכון אוטומטי

הדשבורד מתעדכן אוטומטית בכל הגשת סקר חדשה (קריאה ל-`updateDashboard()` בסוף `doPost`). אם הגשת הסקר נשמרת בהצלחה אבל עדכון הדשבורד נכשל מסיבה כלשהי, התשובה עדיין נשמרת כרגיל — הדשבורד לא יכול לחסום שמירת נתונים.

> **הערת ביצועים:** עדכון הדשבורד קורא ומחשב מחדש את כל גיליון התשובות בכל הגשה. בנפח תשובות גדול (אלפי שורות) זה עלול להאט את זמן השמירה. אם זה קורה, אפשר:
> 1. להסיר את הקריאה ל-`updateDashboard()` מתוך `doPost` ולהריץ אותה רק דרך התפריט הידני (ראו למטה), או
> 2. להגדיר טריגר מבוסס-זמן: בעורך ה-Apps Script לחצו על סמל השעון (Triggers) → **Add Trigger** → Choose function: `updateDashboard` → Select event source: **Time-driven** → בחרו תדירות (לדוגמה כל שעה).

### רענון ידני מתוך Google Sheets

בתוך הגיליון עצמו (לא בעורך ה-Apps Script) מופיע תפריט בשם **"סקר אוטומציה"** עם הפעולה **"רענון דשבורד"** — לחיצה עליה מריצה את `updateDashboard()` ומרעננת את כל הטבלאות והתרשימים מול הנתונים העדכניים בגיליון התשובות.

> התפריט נוצר על-ידי `onOpen()` ומופיע אוטומטית בכל פעם שפותחים את הגיליון בדפדפן (לא נדרשת פריסה נוספת).

## 9. הערות אבטחה

> **קראו לפני הפרסום בסביבת ייצור.**

- **אל תאספו מידע רגיש ביותר** (סיסמאות, מספרי זהות, מידע רפואי) בטופס זה ללא אישור מחלקת אבטחת מידע / IT.
- **אל תעלו קבצים חסויים** לשום שירות שאינו מאושר ארגונית.
- **הנתונים נשמרים ב-Google Sheets** ולא ב-GitHub. וודאו שהגיליון מוגבל לאנשים המורשים בלבד (שיתוף → מי יש לו גישה).
- **ה-Web App URL הוא ציבורי** — כל אחד שיודע אותו יכול לשלוח נתונים. לא מומלץ לסמוך על ה-SHARED_TOKEN כאמצעי אבטחה מלא.
- **לסביבת ייצור בחברה גדולה:** קבלו אישור IT/אבטחת מידע לפני הפצה. שקלו שימוש ב-"Anyone within [your organization]" ב-Web App ובגיליון.
- **השתמשו בחשבון Google ארגוני** ולא בחשבון גוגל פרטי לניהול הגיליון בסביבת ייצור.

## 10. מבנה הפרויקט

```
automation-survey/
├── index.html          # טופס הסקר המלא (9 חלקים)
├── styles.css          # עיצוב RTL עברי
├── script.js           # לוגיקה, וידוא, שליחה ל-Google Script
├── apps-script/
│   └── Code.gs         # Google Apps Script – שמירה ב-Google Sheets
└── README.md           # הוראות הגדרה ופריסה
```

### קבצים שצריך להעלות ל-GitHub Pages:
- `index.html`
- `styles.css`
- `script.js`

**לא צריך להעלות** את תיקיית `apps-script/` — הקוד שם רץ ב-Google Apps Script, לא על השרת.
